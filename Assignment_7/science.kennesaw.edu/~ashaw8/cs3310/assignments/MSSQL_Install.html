Installing MS SQL Server 2008 R2

After you download MS SQL Server 2008 R2 there are multiple steps involved necessary to install it. Run the setup.exe file or the autorun.inf and then take the steps below in the screens that follow:

  1. On this first screen, click on the "Installation" option under the word "Planning":

  2. In the following screen, click on the "New installation or add features to an existing installation" option:

  3. The System Configuration Checker runs a discovery operation on your computer. You can view the details on the screen by clicking Show Details as shown, although you don't have to see the details. After this click on OK to move to the next step. However, if there are any problems identified in this step, you need to address them before moving to the next step:

  4. The Product Key for your installation should be entered for you on the next screen:

  5. Accept the License Terms on the next screen:

  6. Click on the Install button on the next screen:

  7. A pop-up warning will tell you that at the end of the installation a reboot will be required, but the reboot will not happen right now. It will not happen until after all of the installation steps in this tutorial have been completed.

  8. The System Configuration Checker runs a discovery operation on your computer. You can view the details on the screen by clicking Show Details as shown, and that will probably show you a warning about your Firewall setting. You can ignore that warning and click on Next to move to the next step. However, if there are any other problems identified in this step, you need to address them before moving to the next step:

  9. In the next screen, select "SQL Server Feature Installation" and then click Next:

  10. In the next screen, select "Select All" and then click Next:

  11. The System Configuration Checker runs another discovery operation on your computer. You can view the details on the screen by clicking Show Details as shown, although you don't have to see the details. After this click on Next to move to the next step. However, if there are any problems identified in this step, you need to address them before moving to the next step:

  12. Click on the Default Instance option on the next screen and then click on Next:

  13. On the next screen click on Next:

  14. On the next screen, the Server Configuration step, you are to choose what account the SQL Server services will use to do things in the background. I have found that it is best if these services are given network access. To give this access, first click on the "Use the same account for all SQL Server services" button.

    15. Then in the pop-up that shows up next asking for an Account name click on the "Browse" button:

    Then in the "Select User or Group" pop-up that shows up next, type in the word "NETWORK SERVICE" as shown below and then click on OK:

    Then you will go back to the "Use the same account for all SQL Server 2008 R2 services" pop-up, and it should show in the Account Name box "NT AUTHORITY\NETWORK SERVICE", with nothing in the Password box. That is how it should be, so don't put in a password, and just click on the OK button, and then on the Next button after that:

  15. On the next screen, the Database Engine Configuration step, you are to choose who can configure the DBMS. You want to add yourself and the Administrators user group here. To do that, first click ont the Add Current User button:

  16. Your username information should show up (in the example shown, the username is User_2, which probably is NOT what you see because that is probably NOT your username). Next, you want to add the Administrators user group, and to do that, first click on the Add button, and then in the pop-up type in "Administrators" and then click on the Check Names button. After doing this, in front of the "Administrators" name you typed in should be added the name of your computer (which is USER-LAPTOP in the example shown here, but is probably something different for you). Then click on the OK button after that:

  17. Now you should see your Username information and "BUILTIN\Administrators (Administrators)" added as shown below in the SQL Server administrators box. Then click Next:

  18. On the next screen, the Analysis Services Configuration step, follow the exact same instructions that you did in the last step to add your account and the Administrators user group here as well. When you are done you should see your Username information and "BUILTIN\Administrators (Administrators)" added as shown below in the SQL Server administrators box. Then click Next:

  19. On the next screen, select the Install the native mode default configuration option and then click on Next:

  20. You don't have to select anything on the next screen. Just click on Next:

  21. The System Configuration Checker will run one more set of rules to validate your computer configuration with the SQL Server features that you have specified. You can view the details on the screen by clicking Show Details as shown, although you don't have to see the details. After this click on Next to move to the next step. However, if there are any problems identified in this step, you need to address them before moving to the next step:

  22. Now you are almost done with all of the configuration information. The system is ready to be installed. Click on the Install button and be prepared to wait a while as MS SQL Server 2008 R2 is installed on your computer:

  23. You have now successfully installed everything and there will need to be a restart. Click OK to this notification:

  24. More of the same information is given. Click OK to this notification and then click on the Close button and restart your computer:

  25. After the restart you should be all set with Microsoft SQL Server 2008 R2 installed on your computer. When you open the program group, you will run SQL Server Management Studio to complete your assignments.
